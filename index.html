Lulia — an AI teaching partner that replaces the “Teachers Pay Teachers” grind

Teachers burn hours each week building lesson plans and hunting worksheets on marketplaces. Lulia ingests a teacher's curriculum and auto-generates standards-aligned lessons, scaffolded worksheets, activities, games, and even narrated videos — with accommodations baked in (IEP / 504 / ELL / Gifted) and a generation history that refuses to repeat itself.

PROBLEM

Fragmented content, $200K+ district spend on third-party marketplaces, and no way to guarantee standards alignment or non-repetition across a school year.

APPROACH

Five CrewAI crews (16 agents) orchestrate pedagogy, visuals, and QA. A three-tier standards router (Custom › State › National) does dense retrieval over 1.21M standards in pgvector with LLM judgment.

OUTCOME

Pre-production on AWS (ECS Fargate + RDS + S3 + SQS + Bedrock). 26 shipped phases, 22 content templates, 7 design themes, 15/15 tests green, CHANGELOG-disciplined.

Architecture highlights

  • Durable execution (Inngest): 8 Inngest functions orchestrate plan approval, video upload processing (8 checkpointed steps), clip generation (Veo 3), and video crew pipelines — with step-level memoization, declarative concurrency limits, and automatic retries. Replaced all daemon threads. 3 cron jobs handle stale-game cleanup, analytics rollup, and webhook purge.
  • Multi-agent orchestration: 5 CrewAI crews — Curriculum, Lesson, Assets, Video, QA — each with specialist agents, now triggered via Inngest events.
  • Multi-LLM routing: Claude for reasoning, Gemini 2.5 Flash for Slides/Forms/video, Haiku for classification, AWS Bedrock for embeddings. Fallback policies on every call.
  • Deterministic overlays: SHA-256 fingerprinting of prior generations prevents repetition; template + theme system guarantees visual consistency.
  • Video pipeline: 8-step Inngest workflow: ffprobe → thumbnail → AWS Transcribe (polled) → transcript embedding into pgvector → Haiku classification → standards alignment → persist. Concurrency-limited to 3.
  • Real-time layer: FastAPI WebSockets + Redis power live classroom games.
  • Media pipeline: Replicate Flux for avatars, Polly + ElevenLabs TTS, ffmpeg video assembly, Veo 3 Fast clips.
  • Google integration: OAuth into Classroom, Slides, Forms, Drive, Calendar.

Civix — turning fragmented ordinances into defensible answers

Property owners, contractors, and small businesses can't get a straight answer on zoning, permitting, or animal regulation without hiring a lawyer. Civix scrapes and normalizes local ordinances across jurisdictions and runs them through a hybrid system: LLMs handle natural-language intake, a deterministic rules engine (PROHIBITED › RESTRICTED › CONDITIONAL › ALLOWED) produces the actual answer.

WHY HYBRID

LLMs are great at intake and disambiguation. They're not great at "will this decision hold up." A deterministic layer gives reproducibility and audit trails customers can cite.

DATA PIPELINE

Ingests via Municode API, ScrapingBee, and AmLegal; embeds into a RAG store; layers jurisdiction-specific rulesets. 20+ Prisma models map zoning, permits, scrape jobs, and rules.

BUSINESS MODEL

Free tier builds trust with plain-language answers; paid tier sells compliance-ready PDF reports. Stripe + NextAuth + JWT.

Engineering highlights

  • Multi-provider AI routing: Gemini primary, Claude + OpenAI fallbacks.
  • AWS Fargate migration completed — documented in a runbook.
  • GitHub Actions CI: lint, type-check, Jest.
  • Seeded sample data for Cincinnati metro; ready to expand jurisdiction by jurisdiction.
  • Built-in /dashboard/tester harness for rules-engine QA.

SchoolPilot — unifying three K-12 products into one platform

Schools juggle separate tools for classroom monitoring, hall passes, and student dismissal. I built all three (ClassPilot, PassPilot, GoPilot) and then consolidated them under a single tenant-aware SaaS, SchoolPilot — with unified auth, tiered per-product pricing, shared real-time infra, and an integrated admin.

Security & compliance program

When you sell to schools, security isn't a nice-to-have — it's gate one of every procurement process. I built the security posture alongside the product, not as an afterthought.

WISP
Written Information Security Program

Annual review cycle with triggers for material system changes, security incidents, and regulatory shifts. Data classification (Restricted / Confidential / Public), 72-hour incident notification SLA, and severity-tiered response (Critical → pager, High → same-day, Medium → 7-day, Low → 30-day).

HECVAT Lite
Vendor security self-assessment

Completed for EdTech procurement. References iKeepSafe FERPA/COPPA certification (pending). Covers "school official" status under FERPA, COPPA school-consent exception, no-data-mining clause, and prohibition on student data for advertising or AI training.

Security monitor
Real-time threat detection

Deterministic rule engine (no LLM) running every 5 minutes: failed auth spikes, bulk student writes, off-hours admin bursts, cross-school access. Results land in security_events with severity + resolution audit trails. 30-min dedup prevents alert spam.

Authentication & encryption
  • Dual auth: Bearer JWT + session cookies (7-day rolling, 1-hr admin idle timeout)
  • RBAC: admin, school_admin, teacher, office_staff, parent, super_admin
  • Account lockout (10 failures / 15 min) + IP rate limiting (15 / 15 min)
  • TLS 1.2+, HSTS, AES-256 at rest (AWS), AES-GCM for OAuth tokens
  • MFA required for super_admin + AWS production access
Audit & hardening
  • audit_logs table: all admin actions, 2-year retention, indexed for efficient query
  • Helmet.js (CSP, X-Frame, X-Content-Type, Referrer-Policy), CSRF tokens
  • Parameterized queries via Drizzle ORM (SQL injection prevention)
  • Network isolation: ECS in private VPC, RDS not publicly accessible
  • npm audit on critical-level blocks production deploys
